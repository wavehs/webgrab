【云脉互联】 客户端接入教程

5月31日修改

云脉互联（以下简称“云脉AI”）致力于为开发者提供稳定、高效的一站式模型调用服务，让你无需分别接入多个厂商，也能轻松使用 GPT、Claude、Gemini 等主流模型。本教程将从零开始，带你完成账号注册、额度兑换、API Key
获取以及各种主流常见的客户端配置的完整流程，即使是新手，也可以在短时间内快速上手，真正实现“开箱即用”。
📘使用前请仔细阅读完整的使用教程，以确保正确操作并避免不必要的问题。感谢您的配合！
🏆
重要提示：云脉AI国内直连，无需使用任何科学上网工具或代理，否则可能导致连接失败或输出被截
断，请直接用正常网络访问即可。

▶️机制说明

为了方便用户理解平台的服务方式，简单说明一下 API 转发的工作机制。
当用户调用模型时，请求会先发送到云脉AI平台服务器，再由服务器转发到对应的官方模型平台（如 OpenAI、Anthropic、Gemini 等），模型处理完成后，结果再通过API平台服务器返回给用户。
需要说明的是：
•
模型始终运行在官方服务器上
•
平台只负责接口转发和网络连接
•
不参与模型训练或计算
通过这种方式，用户可以在一个平台上调用 GPT、Claude、Gemini 等多种模型，无需分别接入不同厂商接口。
▶️使用说明

为保证接口正常调用，请在接入第三方客户端时注意以下几点：
1、关闭代理或科学上网工具
使用本平台 API 时，请关闭代理或科学上网工具，否则可能出现连接异常，例如提示 “请检查您的互联网连接和网络设置”。
2、填写接口地址（Base URL）
在客户端填写平台提供的接口地址。
不同客户端可能不同，例如：Base URL、API Address、服务器接口、代理地址等。
3、填写 API Key（密钥）
在平台控制台生成 API Key，并填写到客户端对应位置。
4、更换旧接口信息
如果之前使用过其他 API 服务，请同时更换 接口地址和 API Key。
5、接口兼容
本平台接口兼容 OpenAI API 标准，大多数支持 OpenAI 的客户端均可直接使用。

▶️连通底层逻辑

代理地址(Base url Api地址)+key+模型全称（model ID）全称三项参数配置方可调用模型成功；

•
Tips：模型全称以我们服务器模型广场中模型名称为准，不支持自定义命名。
