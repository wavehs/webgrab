**WebGrab** — это мощная и гибкая консольная утилита (CLI) на Node.js, предназначенная для автоматического скачивания и конвертации любых веб-страниц во множественные форматы. Она разработана специально для обхода агрессивных клиентских защит (блокировка копирования, запрет выделения текста, всплывающие окна, платные подписки и оверлеи) и работы с авторизованными сессиями из реальных браузеров.

---

## 🚀 Основные возможности

*   🛡️ **Умный обход ограничений (Bypass)**:
    *   Принудительное включение выделения текста и контекстного меню через внедрение пользовательских CSS/JS.
    *   Автоматическое скрытие рекламных баннеров, всплывающих окон согласия на файлы cookie (Cookie Consent) и окон оформления подписки (Paywall).
    *   Удаление прозрачных полноэкранных заглушек, мешающих взаимодействию со страницей.
*   🔄 **Эмуляция реального пользователя**:
    *   Автоматический скроллинг страницы для подгрузки динамического контента (Lazy Loading изображений и скриптов).
    *   Настройка задержек перед экспортом для стабилизации рендеринга.
    *   Возможность тонкой настройки разрешения окна браузера (Viewport).
*   🔑 **Использование сессий и куки (Cookies)**:
    *   Экспорт и использование активных сессий (cookies) из локально установленных браузеров: **Google Chrome, MS Edge, Mozilla Firefox**.
    *   Поддержка передачи cookies из внешних JSON-файлов для парсинга закрытых страниц.
*   📦 **Многоформатный экспорт (одновременно или по отдельности)**:
    *   `pdf` — печатная версия страницы A4.
    *   `html` — точный снимок текущего DOM-дерева после загрузки скриптов.
    *   `mhtml` — веб-архив "всё в одном" (сжатые картинки, шрифты и разметка).
    *   `png` / `jpg` — качественный полностраничный скриншот.
    *   `md` — чистый структурированный текст в разметке Markdown (очищен от навигации, футеров и рекламы).
    *   `txt` — структурированный Plain Text.
*   👥 **Массовая обработка (Bulk Import)**: загрузка целого списка адресов из текстового файла.

## 📋 Системные требования

*   **Node.js** версии `16.0.0` или выше.
*   Установленный Chromium (устанавливается автоматически через Playwright).

## 🛠️ Установка и настройка

1.  **Клонируйте репозиторий**:
    ```bash
    git clone https://github.com/your-username/webgrab.git
    cd webgrab
    ```

2.  **Установите npm-зависимости**:
    ```bash
    npm install
    ```

3.  **Установите браузеры Playwright**:
    ```bash
    npx playwright install chromium firefox
    ```

## 💻 Использование

Запуск осуществляется с помощью консоли:

```bash
node bin/webgrab.js [URL] [параметры]
```

### Доступные форматы экспорта (`-f, --format`)

| Формат | Результат | Примечание |
| :--- | :--- | :--- |
| `pdf` *(по умолчанию)* | PDF-документ | Оптимизирован под формат A4 |
| `html` | HTML-документ | Снимок обработанного браузером DOM-дерева |
| `mhtml` | Веб-архив `.mhtml` | Инкапсулирует стили и изображения (только в Chromium) |
| `png` / `jpg` | Изображение | Полностраничный скриншот страницы |
| `md` | Файл Markdown | Очищен от меню, футеров и баннеров |
| `txt` | Файл Text | Чистый структурированный текст страницы |
| `all` | Все вышеперечисленные | Сохраняет сразу все форматы в указанную папку |

## 💡 Примеры использования

### 1. Быстрый экспорт в PDF
Скачает страницу и сохранит в папку по умолчанию:
```bash
node bin/webgrab.js https://habr.com/ru/articles/700000/
```

### 2. Экспорт во все форматы в отдельную директорию
```bash
node bin/webgrab.js https://example.com -f all -d ./my_exports
```

### 3. Снятие высококачественного скриншота с заданным разрешением
```bash
node bin/webgrab.js https://github.com -f png --viewport 1920x1080 -d ./screenshots
```

### 4. Доступ к закрытому контенту через сессию Chrome
Утилита найдет профиль Chrome в вашей системе и использует его cookies:
> ⚠️ **Важно**: Перед запуском обязательно закройте Google Chrome, чтобы избежать ошибки блокировки профиля.
```bash
node bin/webgrab.js https://trello.com/b/boardId -c chrome -f pdf
```
*(Также поддерживаются параметры `-c edge` и `-c firefox`)*

### 5. Загрузка cookies из JSON-файла
```bash
node bin/webgrab.js https://private-forum.com -c file:cookies.json -f html
```

### 6. Массовый экспорт страниц из файла списка
Создайте файл `urls.txt` со списком URL (каждый адрес на новой строке) и запустите:
```bash
node bin/webgrab.js --list urls.txt -f pdf -d ./archive
```

## ⚙️ Все параметры командной строки

Вы можете вызвать справку утилиты командой `node bin/webgrab.js --help`.

| Опция | Короткий синоним | Описание | По умолчанию |
| :--- | :--- | :--- | :--- |
| `--format` | `-f` | Формат сохранения (`pdf`, `html`, `mhtml`, `png`, `jpg`, `md`, `txt`, `all`) | `pdf` |
| `--dir` | `-d` | Директория для сохранения файлов | `.` (текущая директория) |
| `--output` | `-o` | Желаемое имя файла без расширения | Заголовок веб-страницы |
| `--cookies` | `-c` | Источник сессии (`chrome`, `edge`, `firefox`, `file:путь_к_json`) | — |
| `--list` | — | Путь к текстовому файлу со списком URL для массовой загрузки | — |
| `--viewport` | — | Разрешение окна браузера (в формате `ШиринаxВысота`) | `1920x1080` |
| `--wait` | `-w` | Дополнительное время ожидания стабилизации JS в мс | `0` |
| `--timeout` | — | Максимальное время ожидания загрузки страницы в мс | `30000` |
| `--headed` | — | Запуск браузера в видимом режиме (полезно для обхода ручной капчи) | Выключено (headless) |
| `--no-bypass` | — | Отключить автоматические скрипты разблокировки страниц | `false` (обход включен) |
| `--no-scroll` | — | Отключить автоматическую прокрутку страницы вниз перед экспортом | `false` (прокрутка включена) |
| `--verbose` | — | Включение расширенного логирования для отладки ошибок | Выключено |

## 🏗️ Структура проекта

Проект имеет модульную архитектуру, где CLI-координатор управляет жизненным циклом браузера, а специализированные экспортеры отвечают за сохранение данных.

```text
webgrab/
├── bin/
│   └── webgrab.js               # Исполняемый файл CLI-команды
├── src/
│   ├── cli.js                   # Парсинг аргументов с использованием Commander
│   ├── browser.js               # Логика инициализации Playwright и загрузки страниц
│   ├── cookies.js               # Извлечение cookies из браузеров и JSON файлов
│   ├── protection-bypass.js     # Инъекционные скрипты для обхода защит
│   ├── utils.js                 # Вспомогательные утилиты (логирование, форматирование)
│   └── exporters/               # Модули экспорта в различные форматы
│       ├── html.js              # Экспорт DOM-модели
│       ├── markdown.js          # Очистка разметки и конвертация в MD (Turndown)
│       ├── mhtml.js             # Сохранение архива MHTML через CDP
│       ├── pdf.js               # Рендеринг PDF
│       ├── screenshot.js        # Захват экрана
│       └── text.js              # Выделение текста и очистка от тегов
├── assets/
│   └── banner.png               # Логотип/Баннер проекта
├── package.json                 # Описание проекта и его зависимостей
└── README.md                    # Документация проекта (этот файл)
```

## 🛠️ Как это работает под капотом?

1.  **Инициализация (cli.js)**: CLI считывает введенные параметры и валидирует URL-адреса.
2.  **Запуск браузера (browser.js)**: Playwright запускает Chromium/Firefox. В зависимости от настроек, подключаются файлы профиля из системных папок браузеров для восстановления сессий.
3.  **Снятие защит (protection-bypass.js)**: 
    *   До загрузки страницы регистрируется stealth-профиль (перезапись `navigator.webdriver`).
    *   На уровне документа перехватываются события мыши и клавиатуры, чтобы сбросить блокировки копирования (`selectstart`, `copy`, `contextmenu` и др.).
    *   Применяется CSS-инъекция, которая сбрасывает значения `user-select: none !important`, а также скрывает известные классы и ID оверлеев подписок (paywall) и баннеров согласия на куки.
4.  **Прокрутка страницы (autoScroll)**: Страница циклично прокручивается вниз небольшими шагами для вызова `IntersectionObserver` и подгрузки ленивых картинок (lazy loading), после чего возвращается наверх.
5.  **Экспорт (exporters/)**: Вызывается выбранный плагин сохранения, который формирует итоговый файл.
