WebGrab — это профессиональный инструмент на платформе Node.js, предназначенный для автоматизированного скачивания, обработки и конвертации веб-страниц в различные форматы. Проект поддерживает работу как через консольный интерфейс (CLI), так и через графическую веб-панель (GUI). 

Система спроектирована для обхода комплексных систем защиты контента, блокировок копирования, детектирования автоматизированного ПО, лимитов запросов, капч и платных подписок. Она поддерживает эмуляцию реального пользователя, подмену цифровых отпечатков (fingerprints), импорт авторизованных сессий и распределенное скачивание через прокси-серверы.

---

## Основные возможности

*   **Многоформатный экспорт**:
    *   `pdf` — печатная версия страницы (оптимизирована под A4).
    *   `html` — снимок DOM-дерева после выполнения всех клиентских скриптов.
    *   `mhtml` — единый архив веб-страницы со встроенными ресурсами (стили, изображения, шрифты).
    *   `png` / `jpg` — полностраничные скриншоты высокого разрешения.
    *   `md` — очищенный структурированный текст в формате Markdown.
    *   `txt` — структурированный текст без разметки.

*   **Обход систем защиты и блокировок**:
    *   Разблокировка выделения текста, контекстного меню и копирования через инъекцию CSS/JS.
    *   Автоматическое скрытие перекрывающих элементов: баннеров куки (Cookie Consent), оверлеев подписок (Paywalls), рекламных модулей и модальных окон.
    *   Деобфускация зашифрованных шрифтов для корректного извлечения текстового контента.
    *   Распознавание текста (OCR) на холстах (canvas-элементах).

*   **Эмуляция пользователя и Stealth-режим**:
    *   Подмена параметров браузера (navigator.webdriver, плагины, языки, экраны) для маскировки под реальное устройство.
    *   Использование профилей устройств (User-Agent, Viewport, сигнатуры для Chrome, Firefox, Safari).
    *   Симуляция человеческого поведения: естественные движения курсора мыши, прокрутка страниц, случайные задержки при вводе и чтении.
    *   Автоматический скроллинг (включая глубокий скроллинг для SPA) для инициализации ленивой загрузки (Lazy Loading).
    *   Раскрытие интерактивных элементов (аккордеонов, спойлеров, details/summary).

*   **Управление сессиями и аутентификацией**:
    *   Прямое извлечение кук (cookies) из локально установленных браузеров (Google Chrome, Microsoft Edge, Mozilla Firefox).
    *   Импорт кук из внешних JSON-файлов.
    *   Сохранение и загрузка рабочих сессий для работы с закрытыми разделами сайтов.

*   **Сетевая инфраструктура**:
    *   Маршрутизация запросов через прокси-серверы с поддержкой списков для автоматической ротации.
    *   Интеграция с сервисами решения капч (2Captcha, Anti-Captcha, CapSolver).
    *   Настраиваемые политики повторных попыток (retries) при сбоях загрузки.
    *   Быстрый режим загрузки (HTTP-клиент без рендеринга) для нетребовательных задач.

*   Node.js версии 16.0.0 или выше.
*   Установленные зависимости браузеров Playwright.

## Установка и настройка

1.  **Клонируйте репозиторий**:
    ```bash
    git clone https://github.com/your-username/webgrab.git
    cd webgrab
    ```

2.  **Установите зависимости**:
    ```bash
    npm install
    ```

3.  **Установите системные зависимости и браузеры**:
    ```bash
    npx playwright install chromium firefox
    ```

## Использование

### Графический интерфейс (GUI)

Запуск веб-панели управления осуществляется командой:
```bash
node bin/webgrab.js --gui
```
После запуска приложение автоматически откроет страницу управления в браузере (по умолчанию `http://localhost:3000`). В GUI доступны настройки параметров обхода, выбор форматов, просмотр логов в реальном времени и менеджер загруженных файлов.

### Консольный интерфейс (CLI)

Базовый синтаксис команды:
```bash
node bin/webgrab.js [URL] [параметры]
```

#### Доступные форматы экспорта (`-f, --format`)

| Формат | Результат | Особенности |
| :--- | :--- | :--- |
| `pdf` | PDF-документ | Выгрузка по умолчанию. Адаптирован под формат печати A4 |
| `html` | HTML-документ | Снимок обработанного браузером DOM-дерева |
| `mhtml` | Веб-архив `.mhtml` | Встраивает все ресурсы страницы (только для Chromium) |
| `png` / `jpg` | Изображение | Полностраничный скриншот страницы |
| `md` | Файл Markdown | Очищен от меню, футеров и баннеров |
| `txt` | Текстовый файл | Чистый структурированный текст страницы |
| `all` | Все доступные | Сохраняет сразу все применимые форматы в целевую директорию |

## Примеры использования CLI

### 1. Сохранение страницы в PDF по умолчанию
```bash
node bin/webgrab.js https://habr.com/ru/articles/700000/
```

### 2. Экспорт во все форматы с сохранением в отдельную папку
```bash
node bin/webgrab.js https://example.com -f all -d ./exports
```

### 3. Скриншот высокого разрешения с имитацией Full HD экрана
```bash
node bin/webgrab.js https://github.com -f png --viewport 1920x1080 -d ./screenshots
```

### 4. Экспорт с использованием авторизованной сессии Google Chrome
Для использования сессии необходимо закрыть инстанс Chrome на компьютере, чтобы избежать блокировки профиля.
```bash
node bin/webgrab.js https://trello.com/b/boardId -c chrome -f pdf
```

### 5. Использование прокси с ротацией и глубоким скроллингом для SPA
```bash
node bin/webgrab.js https://news-portal.com -f html --proxy-list ./proxies.txt --deep-scroll
```

### 6. Запуск скачивания по списку из файла в скрытом режиме с обходом защит
```bash
node bin/webgrab.js --list urls.txt -f md --stealth --humanize -d ./markdown_archive
```

## Все параметры командной строки

Вызов встроенной справки: `node bin/webgrab.js --help`.

| Опция | Альтернатива | Описание | Значение по умолчанию |
| :--- | :--- | :--- | :--- |
| `-f, --format <format>` | — | Формат сохранения (`pdf`, `html`, `mhtml`, `png`, `jpg`, `md`, `txt`, `all`) | `pdf` |
| `-d, --dir <dir>` | — | Целевая директория для сохранения файлов | `.` (текущая папка) |
| `-o, --output <file>` | — | Имя результирующего файла без расширения | Заголовок веб-страницы |
| `-c, --cookies <src>` | — | Источник кук (`chrome`, `edge`, `firefox` или `file:путь_к_json`) | — |
| `-w, --wait <ms>` | — | Задержка перед экспортом после полной загрузки в мс | `0` |
| `--timeout <ms>` | — | Лимит времени ожидания загрузки страницы в мс | `30000` |
| `--viewport <size>` | — | Размер окна браузера в формате `ШиринаxВысота` | `1920x1080` |
| `--headed` | — | Запуск браузера в видимом режиме (для отладки или ручной капчи) | Отключено (headless) |
| `--no-bypass` | — | Отключить автоматические алгоритмы обхода защит страниц | `false` (обход активен) |
| `--no-scroll` | — | Отключить автоматическую прокрутку страницы перед экспортом | `false` (прокрутка активна) |
| `--no-expand` | — | Не раскрывать свернутые интерактивные блоки | `false` (блоки раскрываются) |
| `--deep-scroll` | — | Использовать глубокую прокрутку с ожиданиями для SPA | Отключено |
| `--selector <css>` | — | Извлечь только контент внутри указанного CSS-селектора | Автоопределение |
| `--list <file>` | — | Текстовый файл со списком URL (по одному адресу на строку) | — |
| `--gui` | — | Запуск веб-панели управления в локальном браузере | — |
| `--stealth` | — | Активация режима Stealth (маскировка webdriver, spoofing отпечатков) | Отключено |
| `--profile <name>` | — | Шаблон профиля устройства (`chrome-win`, `chrome-mac`, `firefox-win`, `safari-mac`) | `chrome-win` |
| `--humanize` | — | Имитация человеческих действий (движение мыши, скролл, задержки) | Отключено |
| `--ocr` | — | Распознавание текста на canvas элементах (требует tesseract.js) | Отключено |
| `--decode-fonts` | — | Деобфускация динамических шрифтов (требует opentype.js) | Отключено |
| `--browser-cookies <br>` | — | Быстрый экспорт кук из системного браузера (`chrome`, `firefox`, `edge`) | — |
| `--save-session <path>`| — | Сохранение сессии (куки и state) после загрузки страницы в файл | — |
| `--load-session <path>`| — | Загрузка сохраненной ранее сессии из файла | — |
| `--extra-headers <json>`| — | Дополнительные HTTP-заголовки в формате JSON | — |
| `--proxy <url>` | — | Адрес прокси-сервера (`http://user:pass@host:port`) | — |
| `--proxy-list <path>` | — | Файл со списком прокси-серверов для ротации | — |
| `--captcha-service <n>`| — | Сервис автоматического решения капчи (`2captcha`, `capsolver`, `anticaptcha`) | — |
| `--captcha-key <key>` | — | API-ключ для выбранного сервиса решения капч | — |
| `--retries <n>` | — | Количество попыток повторной загрузки при ошибке | `3` |
| `--no-render` | — | Скачивание через легкий HTTP-клиент (без Playwright, только html/md/txt)| `false` (рендеринг активен) |
| `--verbose` | — | Детальный вывод отладочной информации в консоль | Отключено |

## Архитектура и жизненный цикл запроса

1.  **Считывание конфигурации**: CLI (или GUI через API-запрос) собирает параметры и подготавливает списки прокси, кук и сессий.
2.  **Инициализация движка**: В стандартном режиме инициализируется Playwright с применением модуля `stealth.js` (подмена заголовков, системных переменных) и настроек прокси. При `--no-render` запросы выполняются напрямую через модуль `http-client.js`.
3.  **Авторизация**: Загружаются указанные сессии или системные куки из браузера.
4.  **Загрузка страницы и интерактивные действия**:
    *   Инициализируются обходы защит от копирования на уровне событий окна.
    *   Применяются алгоритмы `behavior.js` для имитации активности пользователя.
    *   Если обнаружена капча и настроен сервис обхода, происходит автоматическая отправка капчи на решение.
    *   Осуществляется скроллинг страницы и раскрытие свернутых элементов.
5.  **Пост-обработка контента**: 
    *   Скрытие всплывающих баннеров, платных оверлеев и рекламных блоков.
    *   При необходимости выполняется OCR-распознавание холстов и расшифровка шрифтов.
6.  **Экспорт результатов**: Вызываются соответствующие экспортеры для записи файлов на диск в выбранных форматах.
